First Day of BEWD

Oct 22nd, 2013

The first day of class was groovy up until I made a typo during our class code-along. It took a moment of my attention to fix. By the time I looked up at the slides and refocused my attention on the instructor we were already a couple of steps ahead… Yikes! I felt lost and frustrated for a while, thankfully Yaritza and Eddie helped me get resituated. I am glad this happened on the first day. It has taught me two things:

Lesson number one: Stop Steven if you are even one typo behind everyone else on a code along. Everything builds. When feelings of lostness and confusion take over your ability to take in new info is close to ‘nil’. Your brain shuts down. Not good…

Lesson number two: If you don’t get something today, ease up on the frustration. Write down where you are stuck and let it go. Revisit it the next day with a rested body, feed tummy, and a fresh set of eyes. It makes a world of a difference! It’s all about brain patterning. Today I reviewed the slides and feel better about them thank goodness. If that’s not the case for you there are always office hours.

Topics covered in our first class:

  • Git
  • Octopress
  • Pushing code to Git Hub Pages

Here are the notes I have based on my understanding (Give me feedback if I am wrong. Sistah wants to learn!):

A. Steps for commiting a file on Git using your command line (CL for short)**

  1. mkdir whatevernameyoufancy – creates a new directory a.k.a. folder, in this case my folder would be called “whatevernameyoufancy”
  2. git init – will initialize a git directory
  3. git add – followed by your file of choice will add it to the repository and will stage it
  4. git status – do this to check what you just staged for commit
  5. git commit -m – followed by your comment in single quotes i.e. git commit -m ‘my blog is finally taking shape here’.
  6. git log – you can do this to see a list all of your commits and logs for your file. To exit the log type q.

B. Steps for creating a new post on Octopress using your CL

  1. rake new_post[“enter post name here”] – this creates the new post. Inside the double quotes enter whatever post name YOU want.
  2. Look at the return value of step 1. It will look something like this “Creating new post: source/posts/2013-10-22-enter-post-name-here-bewd.markdown”. This source/posts/2013-10-22-enter-post-name-here-bewd.markdown is your new file name.
  3. open newfilename.markdown – ie. open source/_posts/2013-10-22-enter-post-name-here-bewd.markdown” This will open the file in your markdown editor from the terminal.
  4. Edit your post
  5. Follow step 2 – 5 from “Steps for commiting a file on Git using your command line” to commit this new file to git. You can do this as many times as you want during your editing process to have as many versions of your file as needed.

C. Steps to deploy your git commits to GitHub

  1. rake generate – converts the file to html so that it is readable on the internet
  2. rake deploy – pushes the code to Git Hub

NOTE

If you are in Octopress here is your workflow:

Follow all steps from part B through Part C and voila you are done and you’ve published your files on GitHub ;)****

I encourage your feedback! Let me know if there is a better way to do this or if I have any errors on here. Thanks. I look forward to your comments.

Your brand-spanking, Ruby, newbie, Yeli
